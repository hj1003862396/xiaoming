# 规格文档符合性审查员提示模板

派发规格文档 (spec) 符合性审查子代理 (subagent) 时使用此模板。

**目的：** 验证实施者构建的内容符合请求（不多不少）

```
Task tool (general-purpose):
  description: "Review spec compliance for Task N"
  prompt: |
    你正在审查实施内容是否与其规格文档 (spec) 匹配。

    ## 请求的内容

    [任务需求的完整文字]

    ## 实施者声称构建的内容

    [来自实施者的报告]

    ## 关键：不要相信报告

    实施者完成得快得可疑。他们的报告可能不完整、不准确或过于乐观。你必须独立验证所有内容。

    **绝对不得：**
    - 相信他们关于实施内容的描述
    - 信任他们关于完整性的声明
    - 接受他们对需求的解读

    **务必：**
    - 阅读他们实际编写的代码
    - 逐行将实际实施与需求进行比较
    - 检查他们声称实施但缺失的部分
    - 寻找他们未提及的额外功能

    ## 你的工作

    阅读实施代码并验证：

    **缺失的需求：**
    - 他们是否实施了所有被请求的内容？
    - 是否有他们跳过或遗漏的需求？
    - 他们声称某个功能有效但实际上没有实施吗？

    **额外/不需要的工作：**
    - 他们是否构建了未被请求的内容？
    - 他们是否过度设计或添加了不必要的功能？
    - 他们是否添加了规格文档 (spec) 中没有的"锦上添花"？

    **误解：**
    - 他们是否以不同于预期的方式解读了需求？
    - 他们是否解决了错误的问题？
    - 他们是否实施了正确的功能但方式错误？

    **通过阅读代码来验证，而非相信报告。**

    汇报：
    - ✅ 符合规格文档 (spec)（如果代码检查后一切匹配）
    - ❌ 发现问题：[具体列出缺失或额外的内容，附文件:行号引用]
```
