# 移除撰写计划中的单元测试要求设计规格文档

**目标：** 从 `xiaoming-writing-plans` 技能中完全移除单元测试与 TDD（测试驱动开发）的强制要求，将其变更为“编写代码 + 命令行/脚本验证”的模式。

## 变更背景

当前的 `xiaoming-writing-plans` 技能强制在生成的实施计划中为每一个任务包含 TDD（测试驱动开发）相关的步骤（如“写失败测试”、“运行测试验证其失败”、“实施让测试通过的最少代码”、“运行测试验证其通过”）。在实际开发过程中，有些项目并不需要强制的自动化单元测试。因此，需要将其简化为更加灵活且不需要编写单元测试的“代码编写 + 命令行验证”模式。

## 详细变更方案

### 修改文件：`skills/xiaoming-writing-plans/SKILL.md`

#### 1. 概述修改
*   **修改前**：
    > 每个任务要修改哪些文件、代码、测试、可能需要查阅的文档、如何测试。将整个计划给他们，分解为细粒度的任务。DRY（不要重复自己）、YAGNI（你不会需要它）、TDD（测试驱动开发）。
*   **修改后**：
    > 每个任务要修改哪些文件、代码、可能需要查阅的文档、如何验证。将整个计划给他们，分解为细粒度的任务。DRY（不要重复自己）、YAGNI（你不会需要它）。

#### 2. 细粒度任务粒度修改
*   **修改前**：
    > **每个步骤是一个动作（2-5 分钟）：**
    > - "写失败测试" — 一个步骤
    > - "运行它确保失败" — 一个步骤
    > - "实施让测试通过的最少代码" — 一个步骤
    > - "运行测试确保通过" — 一个步骤
*   **修改后**：
    > **每个步骤是一个动作（2-5 分钟）：**
    > - "编写最少实施代码" — 一个步骤
    > - "运行命令/脚本验证实现效果" — 一个步骤

#### 3. 任务模板结构修改
*   **修改前**：
    > **文件**：
    > - 创建：`exact/path/to/file.py`
    > - 修改：`exact/path/to/existing.py:123-145`
    > - 测试：`tests/exact/path/to/test.py`
    >
    > - [ ] **步骤 1：写失败测试**
    > ... （TDD 四步法）
*   **修改后**：
    > **文件**：
    > - 创建：`exact/path/to/file.py`
    > - 修改：`exact/path/to/existing.py:123-145`
    >
    > - [ ] **步骤 1：写最少实施代码**
    >
    > ```python
    > def function(input):
    >     return expected
    > ```
    >
    > - [ ] **步骤 2：运行命令验证其正确性**
    >
    > 运行：`python -c "from path.file import function; print(function(input))"`
    > 预期：输出 `expected`

#### 4. 注意事项修改
*   **修改前**：
    > - DRY、YAGNI、TDD
*   **修改后**：
    > - DRY、YAGNI

## 验证计划

通过检查修改后的 `skills/xiaoming-writing-plans/SKILL.md`，确认所有含有“TDD”、“单元测试”、“测试文件”、“写测试步骤”等表述已经被干净地移除或替换。
